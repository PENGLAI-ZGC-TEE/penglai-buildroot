# Buildroot NEMU Flow

This repository now has a first-pass Buildroot integration for the `nemu` path.

## Layout

- `buildroot/`: expected Buildroot source tree
- `br2-external/`: Penglai Buildroot integration tree, currently focused on NEMU
- `br2-external/configs/zgc_tee_nanhuv3a_nemu_defconfig`: default NEMU Buildroot config
- `br2-external/board/nanhuv3a/nemu`: rootfs overlay and post-build logic
- `br2-external/package`: packages migrated out of `riscv-rootfs`

## Commands

```bash
make buildroot-nemu-defconfig
make buildroot-nemu
make run-buildroot
```

## Current scope

- Reuses the existing `opensbi` and `debian-linux-kernel` repositories through Buildroot custom git settings.
- Reuses source code from `riscv-rootfs/apps` through local Buildroot packages during the migration phase.
- Generates the OpenSBI DTB for NEMU from `br2-external/board/nanhuv3a/nemu/system.dts` before invoking Buildroot.

## Migration status

The old `riscv-rootfs` tree is no longer the intended rootfs build entry for the NEMU path, but its source code is still referenced by the first-pass Buildroot packages. Once the Buildroot flow is validated, those remaining sources can be moved fully under `br2-external/package`.

Buildroot now uses local source overrides from `output/nemu/local.mk`, generated by the top-level `Makefile`, so Linux and OpenSBI are built from the workspace `debian-linux-kernel/` and `opensbi/` trees.
